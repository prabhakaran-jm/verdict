## Case Szechuan — Investigation Summary

**Two machines investigated**: DESKTOP-SDN1RPT (workstation) and DC01 (domain controller, C137.local). The DC01 C: partition was inaccessible due to a tool partition-offset limitation, but the 
workstation and its event logs yielded a complete kill chain.

### Kill Chain Reconstruction

**Initial Access / Early Activity**
Domain users `C137\ricksanchez` and `C137\mortysmith` had multiple interactive (VM console) sessions on DESKTOP-SDN1RPT between 22:44–23:10 UTC on 2020-09-18 (F-006). DC01 was already compromised 
at this point by unknown means (the DC investigation was blocked by disk image partition offset constraints).

**Lateral Movement — DC → Workstation**
At **2020-09-19T03:36:22Z**, the domain Administrator account from DC01 (10.42.85.10) performed a Kerberos Type-3 network logon to the workstation. 20 seconds later (03:36:42), the Administrator's
Recycle Bin folder was created — confirming first-ever SMB filesystem access via ADMIN$ (F-005, F-011).

**Execution — Malware Delivery**
At **03:40:00Z**, `coreupdater.exe` (7 KB PE64 with a forged 2010 compile timestamp) was written to `C:\Windows\System32\` via the SMB connection (F-004, F-010). The process ran (PID 8324, PPID 
4008 — already exited) for 2.5 minutes until 03:43:10. At the exact second it exited, a new SYSTEM RSA cryptographic key was created — suggesting it provisioned crypto material before terminating
